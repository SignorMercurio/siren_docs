<Steps>
<Step>

### 设置 OSS 访问凭证（可选）

如果希望将信息收集结果上传到 OSS 并自动在 MkDocs 中展示，需要在运行前[设置 OSS 访问凭证](/credential#设置-oss-访问凭证)。

</Step>
<Step>

### 开始自动化信息收集

```bash tab="本地模式"
./siren recon
```

```bash tab="远程模式"
>>> recon <Client ID>
```

信息收集过程中执行的每一条命令默认均有 30s 超时时间（可通过[配置文件](/config#客户端配置)调整）。收集完成后，会在同目录下保存一份 Markdown 文件。如已配置 OSS 访问凭证，该文件会自动上传至 OSS。

</Step>
<Step>

### 查看数据

<Callout title="关于数据展示" type="info">
  自动 MkDocs 展示功能需自行设置基础设施，详见[关联基础设施](/infra)。
</Callout>

输出的 Markdown 可使用任何 Markdown 阅读器/编辑器打开，同时内部环境也提供 [MkDocs 站点](http://8.217.46.101:8000/) 方便阅读和查阅输出结果：

![mkdocs](../img/mkdocs.png)

建议使用右上角搜索框对信息进行检索。此外，可以点击 AI Summary 展开折叠块，对于该小节内容进行 AI 分析（仅供参考，注意幻觉问题）：

![ai_summary](../img/ai_summary.webp)

<Callout title="文档同步延迟" type="info">
  内部环境中，文档可能需要一小段时间同步到服务器以在 MkDocs 中展示，通常不超过 1 分钟。
</Callout>

</Step>
</Steps>

## 信息收集模块

所有模块均可通过[配置文件](/config#客户端配置)独立开关。

<Accordions>
  <Accordion title="Linux 平台模块">

| 模块 | 配置项 | 采集内容 |
| --- | --- | --- |
| 基础信息 | `recon.basic` | 主机名、操作系统、内核版本、CPU/内存/磁盘使用率、系统运行时间 |
| 用户 | `recon.user` | 用户列表、可登录用户、sudoers 配置、近期登录记录、当前登录用户、影子文件 |
| SSH | `recon.ssh` | SSH 配置、authorized_keys、known_hosts、SSH 密钥文件 |
| 进程 | `recon.process` | 完整进程列表、CPU/内存 Top 20、隐藏进程检测、/proc 异常检查 |
| 网络 | `recon.network` | 网络连接、监听端口、DNS 配置、路由表、ARP 缓存、iptables 规则、网卡混杂模式 |
| 计划任务 | `recon.cron` | 系统级与用户级 crontab、/etc/cron.* 目录、at 任务 |
| 服务 | `recon.service` | systemd 服务、init.d 脚本、开机自启项、rc.local |
| 环境变量 | `recon.env` | 系统环境变量、LD_PRELOAD、LD_LIBRARY_PATH 等敏感变量 |
| 文件 | `recon.file` | 近期变更文件、SUID/SGID 文件、大文件、隐藏文件、敏感目录扫描 |
| Rootkit | `recon.rootkit` | 内核模块列表及异常检测、LD_PRELOAD 检查、系统命令完整性 |
| Webshell | `recon.webshell` | Web 目录下的可疑文件扫描（PHP/JSP/ASP 等） |
| 应用 | `recon.application` | 已安装 RPM/DEB 包、运行中的 Web 应用及中间件 |

  </Accordion>
  <Accordion title="Windows 平台模块">

| 模块 | 配置项 | 采集内容 |
| --- | --- | --- |
| 基础信息 | `recon.basic` | 主机名、操作系统版本、CPU/内存/磁盘使用率 |
| 用户 | `recon.user` | 本地用户列表、管理员组成员、近期登录记录 |
| 进程 | `recon.process` | 完整进程列表、CPU/内存 Top 20、可疑进程检测 |
| 网络 | `recon.network` | 网络连接、监听端口、DNS 配置、路由表、ARP 缓存、防火墙规则 |
| 计划任务 | `recon.schtask` | Windows 计划任务列表 |
| 环境变量 | `recon.env` | 系统环境变量 |
| 文件 | `recon.file` | 近期变更文件、可疑文件扫描 |
| Webshell | `recon.webshell` | Web 目录下的可疑文件扫描 |
| 应用 | `recon.application` | 已安装软件、运行中的 Web 应用及中间件 |

  </Accordion>
</Accordions>
