## 部署与升级

<Accordions>

<Accordion title="如何升级 SIREN？">

SIREN 客户端与服务端均支持一键版本升级。服务端启动时会自动检查更新并提示。

```bash tab="客户端"
./siren upgrade
```

```bash tab="服务端"
./siren_server upgrade
```

添加 `-y` 参数可跳过确认提示，直接升级。

</Accordion>

<Accordion title="启动时报错：Error reading config file">

SIREN 在默认查找路径下未能找到配置文件。请确认以下路径之一存在配置文件：

- 当前目录下的 `config.yaml`（客户端）或 `server_config.yaml`（服务端）
- `./config/` 目录下的 `config.yaml` 或 `server_config.yaml`

也可以通过 `-c` 参数指定配置文件路径：

```bash
./siren recon -c /path/to/config.yaml
```

详见[配置文件](/config)。

</Accordion>

<Accordion title="远程模式下客户端无法连接到服务端？">

请按以下顺序排查：

1. 确认服务端已启动且正在监听指定端口
2. 确认受害主机能够访问控制主机的 IP 和端口（防火墙 / 安全组规则）
3. 确认客户端配置文件中的 `server` 和 `port` 与服务端配置一致
4. 确认服务端 TLS 证书文件存在且路径正确

</Accordion>

</Accordions>

## 使用相关

<Accordions>

<Accordion title="SIREN 与云安全中心 Agent 是否兼容？">

SIREN 与云安全中心 Agent 兼容性良好，但使用过程中可能会触发告警：

1. 使用远程 Shell 功能时，可能触发 `异常的伪终端Shell创建行为` 告警、`反弹Shell` 告警，但不会被拦截。
2. 使用自动化信息收集功能时，可能触发 `蠕虫病毒命令` 告警，但不会被拦截。

以上告警均为预期行为，可在安全中心中标记为误报。

</Accordion>

<Accordion title="如何在容器环境中使用 SIREN？">

部署在**宿主机**上的 SIREN 可以直接收集容器内恶意文件、进程、网络连接等从宿主机层面可观测的项。

以下信息需要通过在**容器内部**启动 SIREN 来收集：
- 命令执行历史
- 计划任务
- 环境变量
- 容器内的文件系统变更

</Accordion>

<Accordion title="远程模式服务端有哪些快捷键？">

SIREN Server 基于 readline 实现，支持标准的 [readline 快捷键](https://github.com/ergochat/readline/blob/master/docs/shortcut.md)，常用快捷键：

- `Tab` — 命令补全
- `Ctrl+A` / `Ctrl+E` — 移到行首 / 行尾
- `Ctrl+W` — 删除前一个单词
- `Ctrl+U` — 清除当前行
- `Ctrl+R` — 搜索历史命令

</Accordion>

<Accordion title="信息收集报告中如何减少误报？">

通过[配置文件](/config#客户端配置)中的 `rules` 字段，可以添加白名单和黑名单规则来过滤已知正常项或高亮可疑项。例如：

```yaml
rules:
  IPWhitelist: 140\.205\.11\.\d+|10\.0\.0\.\d+
  ProcessWhitelist: sshd|systemd|crond
```

规则值支持正则表达式。

</Accordion>

<Accordion title="如何管理插件？">

SIREN 支持通过命令行管理插件的安装、卸载、更新和启停：

```bash tab="本地模式"
./siren plugins install <name>
./siren plugins uninstall <name>
./siren plugins update <name>
./siren plugins toggle <name>
```

```bash tab="远程模式"
>>> plugins <Client ID> install <name>
>>> plugins <Client ID> uninstall <name>
```

插件从配置文件中指定的 `plugins.marketplace` 地址下载。

</Accordion>

</Accordions>
