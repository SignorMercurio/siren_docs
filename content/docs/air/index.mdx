由 AI Agent 自动化实施应急响应，仅需提供受害主机 Client ID、以及安全告警对应的阿里云 UID 和云安全中心 Event ID。

<Callout title="前置条件" type="warn">
  使用前需确保：
  - 已在服务端配置文件中设置 `air.cliPath`（大模型 CLI 工具路径），详见[配置文件](/config#服务端配置)
  - 已设置 [SOAR 平台访问凭证](/credential#设置-soar-平台访问凭证)
  - 控制主机已安装 tmux
</Callout>

## 使用方式

```bash title="SIREN Server"
>>> air <Client ID> <Alibaba Cloud UID> <Security Center Event ID>
```

服务端会通过 tmux 启动一个新面板并自动运行配置的大模型 CLI（e.g. Claude Code）并提供预定义的 prompt 作为输入：

![air](../../img/air.webp)

<Callout title="Tips" type="idea">
  建议结合应急响应 Skills 使用，避免直接使用复杂的 prompt。
</Callout>

## 工作流

1. SIREN 从 SOAR 平台获取安全告警详情
2. 将告警信息和预定义 prompt 传递给大模型 CLI
3. AI Agent 通过 [MCP 集成](/mcp) 自动调用 SIREN 提供的工具
4. Agent 在受害主机上执行信息收集与分析
5. 最终生成完整的应急响应报告

报告效果详见 [示例报告](/air/example)。

除了通过 `air` 命令使用大模型 CLI 执行应急响应，也可以直接使用任意支持 MCP 的 AI 服务达到相同效果，详见 [MCP 集成](/mcp)。
