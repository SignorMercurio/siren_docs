将受害主机上的重要文件上传至 OSS 进行证据留存，常见使用场景：

- 保存重要日志文件（如 Web 访问日志、系统日志）
- 保存恶意样本供后续分析
- 保存配置文件快照

## 设置 OSS 访问凭证

详见 [设置 OSS 访问凭证](/credential#设置-oss-访问凭证)。

## 上传文件

```bash tab="本地模式"
./siren upload <Client file path>
```

```bash tab="远程模式-SIREN Server"
>>> upload <Client ID> <Client file path>
```

上传完成后 OSS 文件路径默认为 `tmp/siren_upload/<file name>`，可通过[配置文件](/config#客户端配置)中的 `oss.objectPath` 自定义上传路径。
