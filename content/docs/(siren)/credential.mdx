<Callout title="安全建议" type="warn">
  推荐使用远程模式并由 SIREN Server 设置凭证，避免在受害主机上直接操作敏感信息。
</Callout>

## 设置 OSS 访问凭证

```bash tab="远程模式-SIREN Server（推荐）"
>>> env OSS_ACCESS_KEY_ID=<YOUR_ACCESS_KEY_ID>
>>> env OSS_ACCESS_KEY_SECRET=<YOUR_ACCESS_KEY_SECRET>
```

```bash tab="本地模式-Linux"
export HISTCONTROL=ignoreboth
 export OSS_ACCESS_KEY_ID=<YOUR_ACCESS_KEY_ID>
 export OSS_ACCESS_KEY_SECRET=<YOUR_ACCESS_KEY_SECRET>
```

```powershell tab="本地模式-Windows"
$env:OSS_ACCESS_KEY_ID="<YOUR_ACCESS_KEY_ID>"
$env:OSS_ACCESS_KEY_SECRET="<YOUR_ACCESS_KEY_SECRET>"
```

<Callout title="关于本地模式 Linux 命令格式" type="info">
  本地模式 Linux 命令中以空格开头，配合 `HISTCONTROL=ignoreboth` 可确保命令历史中不记录敏感信息。Windows 下不支持此特性。
</Callout>

<Accordions>
  <Accordion title="OSS 凭证授权策略">
    建议遵循最小权限原则，仅授予 OSS 访问凭证必要的权限，例如：
    ```json
    {
      "Version": "1",
      "Statement": [
        {
          "Effect": "Allow",
          "Action": "oss:PutObject",
          "Resource": "acs:oss:oss-<Bucket Region>:<Alibaba Cloud UID>:<Bucket Name>/*"
        }
      ]
    }
    ```
  </Accordion>
</Accordions>

## 设置威胁情报 API Key

```bash tab="远程模式-SIREN Server（推荐）"
>>> env TI_API_KEY=<YOUR_TI_API_KEY>
```

```bash tab="本地模式-Linux"
export HISTCONTROL=ignoreboth
 export TI_API_KEY=<YOUR_TI_API_KEY>
```

```powershell tab="本地模式-Windows"
$env:TI_API_KEY="<YOUR_TI_API_KEY>"
```

## 设置 SOAR 平台访问凭证

```bash tab="通过 SIREN Server 配置（推荐）"
>>> env ALIBABA_CLOUD_ACCESS_KEY_ID=<Alibaba Cloud Access Key ID>
>>> env ALIBABA_CLOUD_ACCESS_KEY_SECRET=<Alibaba Cloud Access Key Secret>
```

```bash tab="通过控制主机 Shell 配置"
export HISTCONTROL=ignoreboth
 export ALIBABA_CLOUD_ACCESS_KEY_ID=<Alibaba Cloud Access Key ID>
 export ALIBABA_CLOUD_ACCESS_KEY_SECRET=<Alibaba Cloud Access Key Secret>
```
